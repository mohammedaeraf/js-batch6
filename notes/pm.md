# Managing Software Projects using SCRUM

## 1️⃣ Introduction to SCRUM

### 🎯 Objective

*why SCRUM exists*

### Key Points:

* SCRUM = Agile framework for managing work in iterations (Sprints)
* Focus on:

  * Incremental delivery
  * Continuous feedback
  * Flexibility

### Simple Explanation:

> Instead of building everything at once → we build in **small chunks (Sprints)**.

---

# 2️⃣ SCRUM Roles & Concepts (5 mins)

### Roles:

* **Product Owner (Client / You)**

  * Defines what needs to be built
* **Scrum Master (You / Lead)**

  * Ensures process is followed
* **Development Team**

  * Executes tasks

---

### Key Terms:

* Product Backlog → All tasks
* Sprint → Time-boxed cycle (1 week)
* Sprint Backlog → Tasks selected for sprint
* Increment → Working output

---

# 3️⃣ SCRUM Workflow (10 mins)

Explain the full cycle visually:

```
Backlog → Sprint Planning → Sprint Execution → Daily Standups → Testing → UAT → Release → Retrospective
```

### Emphasize:

* Work is NOT random
* Everything is planned per Sprint
* No continuous ad-hoc work

---

# 4️⃣ Tool Setup (ClickUp/Jira + Slack) (15 mins)

## 🔹 Task Management (ClickUp / Jira)

### Structure:

* Space / Project → Ecommerce App
* Lists → Sprint 1, Sprint 2
* Tasks → Features / Bugs
* Subtasks → Development steps

---

### Recommended Status Flow:

```
To Do → In Progress → In Review → Accepted → Done
```

---

### Good Practices:

* Each task must have:

  * Description
  * Acceptance criteria
  * Assignee
  * Due date

---

## 🔹 Communication (Slack / WhatsApp)

### Channels:

* #dev-team
* #client-updates
* #issues

### Rules:

* No random DMs for project work
* Important updates → channel
* Daily standup updates → channel

---

# 5️⃣ SCRUM Activities Deep Dive (20 mins)

---

## 🔹 5.1 Sprint Planning (5 mins)

### What happens:

* Choose tasks from backlog
* Estimate effort
* Assign tasks

### Output:

✔ Sprint Backlog ready

---

## 🔹 5.2 Daily Standup (5 mins)

### Each person answers:

1. What did I do yesterday?
2. What will I do today?
3. Any blockers?

### Rules:

* Max 10–15 mins
* No long discussions
* Blockers discussed later

---

## 🔹 5.3 Sprint Execution (3 mins)

* Developers work on assigned tasks
* Update status regularly
* Raise blockers early

---

## 🔹 5.4 Sprint Review / Demo (3 mins)

* Show completed work
* Get feedback
* Validate with client

---

## 🔹 5.5 Sprint Retrospective (4 mins)

### Ask:

* What went well?
* What didn’t go well?
* What can we improve?

👉 This is the **most ignored but most powerful step**

---

# 6️⃣ Key Rules You Must Enforce (Very Important)

### 🚫 Avoid:

* Random tasks outside sprint
* No status updates
* Long-running branches
* No testing

---

### ✅ Enforce:

* Every task → ClickUp
* Status always updated
* Unit testing mandatory
* Code merged regularly
* Sprint discipline
